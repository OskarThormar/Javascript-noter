Fetching in the client vs. fetching in the backend.

Fetching på klienten sker i browseren og bruges til at hente data direkte fra en API eller server.
Fetching i backend sker på serveren og bruges til at hente data fra en ekstern API, database eller en anden server.

Know how to define environment variables natively in Node.js (natively = not using any libraries).

Miljøvariabler bruges til at gemme følsomme oplysninger som API-nøgler, databaser eller konfigurationsindstillinger.
Node.js kan tilgå miljøvariabler via process.env, som indeholder alle systemets variabler.
Variabler kan sættes direkte i terminalen før kørsel af en Node.js-fil.
På Windows bruges set og på macOS/Linux export til at definere dem i terminalen.
Kan også sættes midlertidigt ved at angive dem før en kommando.
I en Node.js-fil kan process.env.VARIABEL_NAVN bruges til at hente værdien.

Can define and run scripts with NPM.

NPM scripts bruges til at automatisere opgaver, som at starte en server, bygge et projekt eller køre tests.
Defineres i package.json under "scripts"-sektionen og kan køres med npm run script-navn.
Standard scripts (start, test) kan køres uden run, fx npm start i stedet for npm run start.
Kan kalde kommandoer som node, nodemon, eslint, eller custom scripts.
Shell-kommandoer kan inkluderes, fx "build": "rm -rf dist && webpack".
Flere scripts kan kædes sammen ved at bruge && eller & (parallel execution).

Structuring HTML with semantic HTML. Knows how to avoid having a footer in the middle of the page.

Semantisk HTML betyder at bruge de korrekte tags, som beskriver sidens indhold og forbedrer tilgængelighed og SEO.
Struktur bør følge en logisk opbygning med header, main, section, article, aside og footer.
En footer bør altid være placeret i bunden af siden, ikke midt i layoutet.
Problemer med en "flyvende" footer opstår ofte, hvis main-indholdet ikke fylder nok eller hvis footer er placeret inde i en anden container.
CSS Flexbox eller Grid kan bruges til at sikre, at footeren forbliver i bunden af siden, selv når der er lidt indhold.
Brug af min-height: 100vh; på body eller main sikrer, at footeren ikke rykker op, hvis der er lidt indhold.

Can point to which files are client files and which are server files.

Klientfiler kører i browseren, mens serverfiler kører i Node.js på serveren.
Klientfiler inkluderer HTML, CSS, JavaScript (frontend) og assets som billeder og skrifttyper.
Serverfiler indeholder backend-kode skrevet i Node.js (fx Express-server, API'er, databaseforbindelser).
Klientfiler kan aldrig interagere direkte med databasen, det håndteres via serveren.
I en fullstack-applikation sender klienten requests til serveren, som behandler dem og sender svar tilbage.

Understands one way to style a footer so that it appears at the bottom, always.

Footeren skal være i bunden af siden, selv når der er lidt indhold.
Problemet opstår ofte, hvis main ikke fylder nok, så footeren rykker op på skærmen.
Kan løses med CSS Flexbox, Grid eller position: fixed;.
flex-grow: 1; bruges til at skubbe footeren ned, når der er lidt indhold.
position: fixed; kan bruges, men gør footeren statisk på skærmen.
