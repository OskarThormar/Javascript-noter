Understands the use case for Nodemon and its limitation for development.

Nodemon genstarter automatisk en Node.js-server, når filer ændres, hvilket sparer tid under udvikling.
Bruges primært i udviklingsmiljøer for at undgå manuel genstart af serveren efter hver ændring.
Installeres globalt eller lokalt og køres med nodemon app.js i stedet for node app.js.
Overvåger kun filændringer i projektmappen, men kan konfigureres til at ignorere eller overvåge bestemte filer.
Understøtter ikke produktionsmiljøer, da den er designet til udvikling og kan forbruge unødvendige ressourcer.
Kan have problemer med caching af moduler, hvilket nogle gange kræver en manuel genstart.
Kan konfigureres via en nodemon.json-fil for at justere adfærd og overvågning af filer.

Can use loop methods (+map, +filter, %reduce). Can choose the right one at the appropriate time:
We wish to use loop methods and in a functional manner (avoiding side effects).
Only use for loops in JavaScript when finger counting.

map bruges til at lave en ny array ved at transformere hvert element.
filter bruges til at fjerne elementer, som ikke opfylder en betingelse.
reduce samler en array til én værdi, fx en sum eller et objekt.
Loop-metoder ændrer ikke den originale array, men returnerer en ny.
Brug funktionel programmering og undgå side effects (ændring af variabler udenfor funktionen).
For-loops bør kun bruges, når man "tæller på fingrene", fx ved simple iterationer uden transformation.
Reducerer behovet for midlertidige variabler, da resultatet gemmes direkte i den nye array.

Understands why a functional approach to loops is to be preferred and how to implement it without side effects.

En funktionel tilgang betyder, at loops ikke ændrer data direkte, men returnerer en ny værdi.
Metoder som map, filter og reduce bruges, fordi de ikke muterer den originale array.
Side effects opstår, når en funktion ændrer variabler udenfor sig selv, hvilket gør koden sværere at debugge.
Reducerer behovet for midlertidige variabler og gør kode mere kompakt.

Can use the right terminology when it comes to the different parts of an URL.

Protocol (fx https://) angiver, hvordan data sendes mellem klient og server.
Domain eller host (fx example.com) er adressen på serveren.
Path (fx /products) viser, hvilken ressource der tilgås på serveren.
Query parameters (fx ?category=shoes&color=blue) bruges til at sende ekstra data til serveren.
Fragment (fx #section1) refererer til en specifik del af en side, men sendes ikke til serveren.
Port (fx :3000) angiver, hvilken port serveren lytter på (standard er 80 for HTTP og 443 for HTTPS).

Can make a POST request and parse it in Express.

En POST request bruges til at sende data til en server, ofte for at oprette nye ressourcer.
Data sendes i request body, i modsætning til GET, hvor data sendes i URL'en.
Express håndterer POST requests med app.post(path, callback).
Middleware som express.json() kræves for at parse JSON-data fra request body.
Dataen kan tilgås via req.body, når den er parset korrekt.
POST bruges typisk med fetch() fra frontend for at sende data til serveren.

Understands the need for body parsing and how to do it in Express.

Body parsing er nødvendigt, fordi data i en POST/PUT request sendes i request body og ikke automatisk kan læses i Express.
Express kan ikke læse request body direkte, så middleware skal bruges til at parse den.
express.json() middleware gør det muligt at parse JSON-data, så det kan tilgås via req.body.
express.urlencoded({ extended: true }) bruges til at parse form-data, som sendes fra HTML-formularer.
Hvis body parsing ikke er aktiveret, vil req.body være undefined.
Forskellige formater kræver forskellige parsere – JSON (express.json()), URL-encoded (express.urlencoded()), eller rå tekst (express.text()).
