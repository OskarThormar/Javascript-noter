Can create a full CRUDable REST API that follows conventions.

CRUD (Create, Read, Update, Delete) er de fire grundlæggende operationer i et REST API.
API'et skal følge REST-konventioner, hvor hver ressourcetype har sin egen URL og bruger de korrekte HTTP-metoder.
Bruger POST til at oprette, GET til at læse, PUT/PATCH til at opdatere og DELETE til at slette data.
Ressourcenavne er altid flertal (fx /users i stedet for /user).
Bruger route-parametre (/users/:id) til at håndtere individuelle elementer.
Data sendes og modtages som JSON, og body parsing er nødvendigt for POST og PUT/PATCH.
API'et skal returnere meningsfulde statuskoder (fx 201 Created, 400 Bad Request, 404 Not Found).

Can get the Date in Javascript.

new Date() opretter et nyt dato-objekt, der repræsenterer den aktuelle dato og tid.
Kan hente specifikke dele af datoen som år, måned, dag, time osv. med metoder som:
getFullYear() → Henter årstallet
getMonth() → Henter måneden (0-baseret, januar = 0)
getDate() → Henter dagen i måneden
getDay() → Henter ugedagen (0 = søndag, 6 = lørdag)
getHours(), getMinutes(), getSeconds() → Henter tidspunkter
Kan formatere datoen som string med toISOString() eller toLocaleDateString().
Tidszoner påvirker datoer, og Date-objektet bruger som standard UTC.

Can write a fetch request without hesitation.

fetch() bruges til at lave HTTP requests til en server for at hente eller sende data.
Returnerer en Promise, så .then() eller async/await bruges til at håndtere svaret.
Kan hente data med GET og sende data med POST, PUT, PATCH eller DELETE.
Kræver headers som "Content-Type": "application/json", når der sendes JSON-data.
Fejlhåndtering er vigtig, da et request kan fejle (brug .catch() eller try/catch).
Kan bruges med REST API’er for at interagere med en database eller ekstern tjeneste.
